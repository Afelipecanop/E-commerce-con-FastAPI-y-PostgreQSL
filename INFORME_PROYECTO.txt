1. DESCRIPCIÓN GENERAL
================================================================================

Proyecto e-commerce en desarrollo activo, con backend desarrollado en FastAPI,
PostgreSQL como motor de base de datos y un frontend en HTML, CSS y JavaScript
vanilla. La plataforma permite gestionar usuarios, catálogo de productos, carrito,
pagos, contenido visual de la tienda y una experiencia de compra más completa
para clientes y administradores.

Estado actual: aproximadamente 95% del MVP funcional.
- Backend: ~95% funcional
- Frontend público: ~92% funcional
- Panel de administración: ~88% funcional
- Integraciones externas: activas y en uso
- Testing automatizado: 0% (sin suite implementada aún)

El proyecto se encuentra en una etapa madura de desarrollo, con módulos principales
operativos y con una base sólida para seguir escalando con nuevas mejoras.


2. PROPÓSITO Y ALCANCE
================================================================================

OBJETIVOS PRINCIPALES:

3. ARQUITECTURA TÉCNICA
================================================================================

3.1 BACKEND - FastAPI + PostgreSQL
------------------------------------

El backend está organizado con un enfoque modular, separando rutas, modelos,
esquemas, middleware y servicios.

BASE DE DATOS:
- users: usuarios, autenticación y roles
- products: catálogo de productos
- carts y cart_items: carrito de compras por usuario
- orders y order_items: órdenes y detalle de compra
- store_layout: bloques dinámicos del layout de la tienda
- product_pages: configuraciones adicionales del contenido de páginas de producto

PRINCIPALES FUNCIONALIDADES BACKEND:
- Rutas de autenticación y perfil de usuario
- CRUD de productos con control de acceso para administradores
- Gestión de carrito con validación de stock
- Creación de órdenes y flujo de checkout
- Confirmación de pagos y actualización del estado de órdenes
- Gestión de bloques del layout con persistencia en base de datos
- Endpoints para generación de contenido con IA
- Endpoints dinámicos para personalizar páginas de producto

SERVICIOS Y CAPAS:
- Auth service: registro, login y generación de tokens
- Middleware: protección de rutas por usuario y por administrador
- Schemas: validación y serialización de datos
- Models: estructura relacional del negocio

3.2 FRONTEND - HTML/CSS/JavaScript
------------------------------------

El frontend está implementado con HTML semántico, estilos propios y JavaScript
vanilla para interactuar con la API.

PÁGINAS IMPLEMENTADAS:
- index.html: catálogo principal
- login.html: registro e inicio de sesión
- cart.html: visualización y gestión del carrito
- checkout.html: resultado del flujo de pago
- admin.html: panel administrativo y editor visual
- product.html: página de detalle de producto con descripción, características,
  especificaciones y productos relacionados
- nosotros.html: página institucional con información de marca y propuesta de valor

COMPONENTES DESTACADOS:
- Diseño responsive
- Integración con la API para carga dinámica de datos
- Persistencia de sesión con token de acceso
- Menú administrativo con vistas de productos y layout
- Experiencia visual más refinada en la página de producto

5. INFRAESTRUCTURA DEL PROYECTO
================================================================================

5.1 ARQUITECTURA DE INFRAESTRUCTURA
------------------------------------

El proyecto está compuesto por tres capas principales:
- Capa de frontend: archivos estáticos HTML/CSS/JS servidos desde un hosting estático.
- Capa de backend: API REST desarrollada con FastAPI y ejecutada mediante Uvicorn.
- Capa de datos: base de datos PostgreSQL gestionada por un servicio relacional.

La comunicación entre el frontend y el backend se realiza mediante peticiones HTTP
seguras, con autenticación por token JWT y configuración de CORS para permitir el
acceso desde entornos locales y de despliegue.

5.2 COMPONENTES DE DESPLIEGUE
------------------------------------

Backend:
- Ejecutado con Uvicorn en un entorno ASGI.
- Configuración de arranque mediante Procfile.
- Versión de runtime definida en runtime.txt para compatibilidad del entorno.
- Endpoints de salud disponibles en /health y /db-check para verificación básica.

Frontend:
- Estructurado como sitio estático con archivos HTML, CSS y JS.
- Diseñado para abrirse directamente desde un navegador o desde un hosting estático.
- Integra la API remota mediante JavaScript y variables de entorno en la lógica del cliente.

Base de datos:
- PostgreSQL como motor principal.
- Modelos relacionales gestionados con SQLAlchemy.
- Migraciones disponibles a través de Alembic.

5.3 VARIABLES Y CONFIGURACIÓN
------------------------------------

La aplicación depende de variables de entorno para su funcionamiento en distintos
entornos. Entre las más importantes se encuentran:
- DATABASE_URL: conexión a la base de datos PostgreSQL
- SECRET_KEY: firma de tokens JWT
- STRIPE_SECRET_KEY: acceso a la API de Stripe
- STRIPE_WEBHOOK_SECRET: verificación de eventos webhook
- FRONTEND_URL: URL base del frontend para redirecciones y CORS
- ANTHROPIC_API_KEY: acceso a la integración de IA

Estas variables deben configurarse de forma segura y no deben exponerse en el código
ni en archivos compartidos públicamente.

5.4 SEGURIDAD E INFRAESTRUCTURA
------------------------------------

La infraestructura actual contempla:
- Protección de endpoints con middleware de autenticación y autorización
- Tokens JWT para sesiones de usuario y administración
- CORS habilitado para orígenes permitidos
- Gestión de secretos vía variables de entorno
- Integración con webhooks para confirmación de pagos de forma asíncrona

La base de infraestructura del proyecto ya está definida y operativa en su estructura
